0.3 👋 Tu primer saludo en Python: ’Hola, mundo

¡Bienvenido al emocionante mundo de la programación! 🎉 Cuando comenzamos a aprender un nuevo lenguaje, es tradición iniciar con un clásico: el famoso ejemplo de “Hola, mundo”.

Por simple que parezca, esta línea de código no solo nos enseñará a imprimir un texto en la salida, sino que también nos permitirá familiarizarnos con la ejecución de código en las celdas de un cuaderno. ¡Así que, manos a la obra y empecemos a programar! 🐍💻

Consejo: Para ejecutar el código de Python en la celda de código a continuación, haz clic en la celda para seleccionarla y presiona Shift + Enter.

1 🎉 ¡Prueba tu primera salida en Python!

Ahora es el momento de poner en práctica lo que hemos aprendido. ¡Vamos a imprimir un mensaje en la consola!

Escribe el siguiente código:

print('¡Hola, Python!')
¡Hola, Python!

Después de ejecutar la celda anterior, deberías ver que Python imprime ¡Hola, Python!.

¡Felicidades por haber ejecutado tu primer código en Python! 🎉 ¡Sigue así y prepárate para explorar más sobre este increíble lenguaje de programación! 🚀

💡 Consejo: La función print() es muy útil. En este caso, le pasaste la cadena de texto ‘¡Hola, Python!’ como argumento, indicando a Python qué es lo que deseas imprimir en la consola. ¡Así de sencillo! 😊

1.0.1 🔍 ¿Cuál es la versión de Python que estamos utilizando?

Hoy en día, hay dos versiones populares del lenguaje de programación Python en uso: Python 2 y Python 3. Sin embargo, la comunidad de Python ha decidido avanzar y dejar atrás Python 2, y muchas bibliotecas populares han anunciado que ya no brindarán soporte para esta versión.

Dado que Python 3 es el futuro, en este curso lo utilizaremos de manera exclusiva. Pero, ¿cómo sabemos que nuestro cuaderno se está ejecutando con un entorno de Python 3? ¡Es fácil! Solo necesitamos mirar en la esquina superior derecha de este cuaderno, donde deberías ver “Python 3”.

También podemos preguntarle directamente a Python y obtener una respuesta detallada. ¡Intenta ejecutar el siguiente código! 

# Verifica la versión de Python

import sys

# Obtener la versión de Python
version = sys.version

# Mensaje
mensaje = f"""
¡Hola! 🎉

Estás utilizando Python {version}.
Esta es una versión poderosa y versátil que te permitirá explorar el mundo de
la programación. 🚀

Recuerda que es importante conocer la versión que estás utilizando, ya que
algunas características pueden variar entre versiones. ¡Prepárate para aprender
y crear cosas increíbles! 💻
"""

print(mensaje)

1.0.2 ✍️ Tus comentarios en Python

Además de escribir código, ¡siempre es una buena idea agregar comentarios! 💬 Los comentarios ayudan a otros a entender lo que intentabas lograr (es decir, la razón por la que escribiste un fragmento de código en particular). No solo esto beneficia a otras personas, sino que también puede servir como un recordatorio para ti cuando regreses a tu código semanas o meses después.

Para escribir comentarios en Python, simplemente utiliza el símbolo de número # antes de tu comentario. Cuando ejecutes tu código, Python ignorará todo lo que esté después del # en esa línea.

Así que no olvides comentar tu código y hacerlo más legible para todos. ¡Es una práctica excelente! 🌟

2 📝 Práctica de escritura de comentarios

# Esta línea imprime una cadena de texto
print('¡Hola, Python!')  # Esta línea imprime una cadena de texto
¡Hola, Python!

Después de ejecutar la celda anterior, notarás que Esta línea imprime una cadena de texto no apareció en la salida, ¡porque era un comentario (y por lo tanto fue ignorado por Python)! 🚫

La segunda línea tampoco se ejecutó, ya que print(‘Hola’) estaba precedida por el símbolo de número (#) también. Dado que esto no es un comentario explicativo del programador, sino una línea de código real, podemos decir que el programador comentó esa segunda línea de código. 📝

2.0.1 🚨 Tipos de errores en Python

Todos cometemos errores. 😅 Para muchos tipos de errores, Python te informará que has cometido un error mostrándote un mensaje de error. Es muy importante leer los mensajes de error con atención para entender realmente dónde cometiste el error y cómo puedes corregirlo. 🔍

Por ejemplo, si escribes print como frint, Python mostrará un mensaje de error. ¡Inténtalo!

# Imprimir una cadena como mensaje de error

frint("¡Hola, Python!")  # ¡Esto provocará un error!

      2 
----> 3 frint("¡Hola, Python!")  # ¡Esto provocará un error!

NameError: name 'frint' is not defined

El mensaje de error te indica:

  1. dónde ocurrió el error (esto es más útil en celdas grandes de notebooks o scripts), y
  2. qué tipo de error fue (en este caso, un NameError).

Aquí, Python intentó ejecutar la función frint, pero no pudo determinar qué es frint ya que no es una función incorporada y tampoco la hemos definido previamente.

Notarás que si cometemos un tipo diferente de error, como olvidar cerrar la cadena de texto, obtendremos un error distinto (es decir, un SyntaxError). ¡Inténtalo a continuación!

2.0.2 ❓ ¿Puede Python detectar tu error antes de ejecutar el código?

Python es lo que se llama un lenguaje interpretado. Los lenguajes compilados examinan tu programa completo en tiempo de compilación y pueden advertirte sobre una clase completa de errores antes de la ejecución. En contraste, Python interpreta tu script línea por línea mientras lo ejecuta.

Python detendrá la ejecución de todo el programa cuando encuentre un error (a menos que el error sea esperado y manejado por el programador, un tema más avanzado que abordaremos más adelante en este curso).
